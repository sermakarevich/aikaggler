# 1. Cross-validation strategy
In this competition, I use a very simple validation strategy: stratified by State and choosing a seed so that the mean and standard deviation of the labels are similar across all 5 folds. There are a few main reasons: first, the dataset is too small, so if we split it in an extreme way, such as grouping by date or by state, the label distribution would become heavily distorted. Second, the host has confirmed that there are no new species in the public and private test sets. Some states have a certain species while others don’t. Grouping by date also risks missing some species or not having all four seasons represented in the training set. As I understand it, each date group corresponds to a trip to a particular farm to collect samples, and it’s possible that some clover species exist only in certain farms.

Basically, my split is almost certainly suffering from data leakage, and I accept that. To mitigate this, I use the CV–LB correlation and the LB score to evaluate the model.
# 2. Synthetic dataset
Right from the start of the competition, I focused on methods for generating additional training data. However, almost all of my initial experiments failed: the synthetic data, when used to train the model, consistently caused the Public Leaderboard score to drop.
One of the main challenges in using synthetic datasets for image regression is the ground truth issue; when an image is modified, the biomass value of each target changes accordingly. To address this, I employed a pseudo-labeling strategy: I used a model trained on the original training set to generate labels for the synthetic dataset, and subsequently trained the final model on the combined dataset (original training set + synthetic dataset).
I used [Qwen Image Edit](https://github.com/QwenLM/Qwen-Image) to create two types of synthetic datasets:
1. The synthetic dataset was generated by simulating seasonal weather changes and adding artifacts such as gravel and rocks.
2. I removed all grass vegetation from the images, retaining only the bare soil, and set the target values for clover, dead, and green to 0. These images were then combined with the training dataset to help the model discriminate between biomass-containing regions and non-biomass regions.

# 5. Model
## Backbone
The high-resolution dense features of DINOv3 are truly remarkable. In density estimation tasks, detailed features with high discriminative power are crucial. I experimented with various backbones, including modern ones like dinov2, XCiT, SigLIP, and PE spatial (Perception Encoder), but both CV and LB scores were significantly lower compared to DINOv3.
Although I wanted to ensure diversity, I could only select backbones from DINOv3 family, where ConvNeXt-Large performed significantly worse than ViT-Large.
## Weakly supervised semantic segmentation
The idea is that instead of treating this merely as a standard image regression problem, I want to approach it as a density estimation problem. Therefore, I used DINOv3 to extract feature maps of size (N, C, H, W). Then, I tried attaching different heads to estimate the density map. The final results would be the target values for clover, dead, and green.
### First attempt: Simple head Conv + ReLU
```python
        self.model = timm.create_model('vit_large_patch16_dinov3_qkvb.lvd1689m', pretrained=True, features_only=True)
        self.pre_pool_conv = nn.Conv2d(feature_channels, feature_channels, kernel_size=3, padding=1, bias=False)
        self.global_pool = nn.AdaptiveAvgPool2d((1, 1))
        self.reg_head_main = nn.Linear(feature_channels, 3)  # clover, dead, green
    def forward(self, x):
        ...
        features = self.model(x)
        last_feature = features[-1]
        last_feature = self.pre_pool_conv(last_feature)
        last_feature = F.relu(last_feature)
        pooled = self.global_pool(last_feature).flatten(1)
        target_values = self.reg_head_main(pooled)
```
### Second attempt: DPT head
I wanted to design the model as a semantic segmentation model and found [SegDino](https://github.com/script-Yang/SegDINO). By utilizing multiple feature maps and adopting the DPT head from SegDino, I designed the model to output a heatmap where each pixel corresponds to the target value of a 16x16 patch in the image.

#### Backbone
- Extracts intermediate features at indices [4, 11, 17, 23].

#### DPT Head
- Projects the 4 input feature maps to [96, 192, 384, 768] channels.
- Refines them with 3x3 convolutions ("Scratch" layers).
- Upsamples lower-resolution features and concatenates them.
- Final 1x1 convolution reduces channels to 3 (for the 3 classes).

#### Dual-Path Strategy:
- The input image is split into left and right halves.
- Each half is processed independently through the backbone and head.
- The results are globally pooled and then summed together.

![](https://www.googleapis.com/download/storage/v1/b/kaggle-user-content/o/inbox%2F2832287%2F5ad2ee7fd6ae4325e25e3aa40c1e53f3%2Fmodel_structure_x3.png?generation=1769840662099620&alt=media)

```python
    ...
    self.model = timm.create_model('vit_large_patch16_dinov3_qkvb.lvd1689m', pretrained=True)
    self.feature_indices = [4, 11, 17, 23]
    self.head = DPTHead(3, self.model.embed_dim, 128, False, out_channels=[96, 192, 384, 768])

    self.IMAGENET_DEFAULT_MEAN = torch.tensor([0.485, 0.456, 0.406]).to(device_id)
    self.IMAGENET_DEFAULT_STD = torch.tensor([0.229, 0.224, 0.225]).to(device_id)

    self.global_pool = nn.AdaptiveAvgPool2d((1, 1))

    def forward(self, x, return_features=False):
        ...
        w = x.shape[3]
        x_l = x[:,:,:,:w//2]
        x_r = x[:,:,:,w//2:]
        outputs_l, features_l = self.model.forward_intermediates(x_l, indices=self.feature_indices)
        outputs_r, features_r = self.model.forward_intermediates(x_r, indices=self.feature_indices)
        features_l = self.head(features_l)
        features_r = self.head(features_r)

        main_logits_l = self.global_pool(features_l).flatten(1)
        main_logits_r = self.global_pool(features_r).flatten(1)
        main_logits = main_logits_l + main_logits_r
```

## Seperate target models
During training, I noticed that the green target usually converged very early and then started to overfit. The RMSE of the green target typically decreased until around epochs 11–15, and then kept increasing. The remaining targets, clover and dead, usually converged later, around epochs 24–30.
Therefore, I used the model at epoch 15 to predict the green target, and the model at epoch 26 to predict the other two targets, clover and dead.

# 8. Trying to find a different approach but without success
1. Decoding-based Regression: https://www.kaggle.com/competitions/csiro-biomass/discussion/668735
2. I attempted to use [SAM 2](https://github.com/facebookresearch/sam2) for segmentation. However, since the clover and grass are extremely small and densely intertwined, effective segmentation using SAM 2 proved to be virtually impossible.
3. I tried using [DiffuseMix](https://github.com/khawar-islam/diffuseMix), but as mentioned earlier, the problem with synthetic data is that when the image changes, the label changes along with it.
4. Used Excess Green index for segmentation, then used the mask as the 4th channel. Public score did not improve.
5. Since the biomass estimation problem relies on plant density and height, Height_Ave_cm was used as an auxiliary loss. I also tried using [Depth Anything v3](https://github.com/ByteDance-Seed/Depth-Anything-3) + DINOv3 ViT-Large in a dual-stream model. All methods decreased the public score.
6. Used Pre_GSHH_NDVI, State, Species, and Height_Ave_cm as auxiliary losses. All resulted in no change or a decrease in the Public score.
7. Pretrained on an external dataset, then fine-tuned on the training dataset; the public score decreased relatively significantly.
8. Test time training adaptation: abandoned because it was too time-consuming during inference.
9. Hflip TTA has no effect or negligible impact with 6-4 weights.
